GAS

  • GAS
    • (def)
      • Google が提供する各種サービスの自動化・連携を行うためのローコード開発ツール。
        • *ローコード開発: 可能な限りソースコードを書かずに開発する手法。
      • ブラウザ上で開発。
      • JS をベースに作られている。一部、GAS 専用の関数が追加されている。
      • API 経由で外部のサービス・アプリなどと連携可能。
      • 注意点：
        • 実行時間に制限がある。（有料 Google Workspace アカウントでも同様）
          • スクリプト実行時間：　６分（１回の実行につき）
          • カスタム関数実行時間：　３０秒（１処理につき）
        • ローカルファイルの操作は行えない。
    • Use Cases
      • *GASを使って開発したプログラムを、Web アプリとして公開することも可能。
      • Google スプレッドシートのデータを Gmail に取り組んでメールを
        自動作成・送信。
      • Google Drive 上のファイルが更新されたことを Gmail / チャットで通知する。
      • Google Forms の自動作成、指定アドレスに入力内容を自動送信。
      • Scraping により収集した情報を Google Spreadsheet に登録。
      • 指定時間に Google Sheets に登録した Slack / Chatwork / 又は会計システムに送信する。
    • 2種類の開発方式
      • *2 types of scripts used with Google Apps
      • Container-Bound scripts
        • *コンテナバインド型
          • Googleサービスに紐づくプログラム開発に使われる方式。
          • Googleサービスと連携可能。Webアプリ開発可能。
        • (doc) Scripts created from a Google Apps (Sheets/Docs/Slides/Forms) file. Said scripts are bound (attached) to its container (a Google Apps file).
          • Cannot be detached from the file they are bound to.
          • Created by selecting Extensions > Apps Script within the Google App.
      • Standalone scripts
        • *スタンドアロン型
          • Googleサービスに紐づかないプログラム開発に使われる方式。
          • Googleサービスと連携可能。Webアプリ開発可能。
        • (doc) Scripts not bound to a Google Apps (Sheets/Docs/Slides/Forms) file.
          • Created via Google Drive / script.google.com

    • Shortcuts
      • Cmd + D → Selects the next occurence of the current selection (highlighted bit).

  • Variables
    • (def)
      • Variable names are case sensitive.
      • camelCase is the convention.
      • First character must not be a number.
      • Can only contain letters/digits, or the symbols $ & _. (No hyphens!)
      • Reserved words: let/class/return/function/etc.
    • var
      • (def)
        • (mdn) Declares function-scoped or globally-scoped variables. Re-declarable.
        • (stack) Difference between var & let: Variables declared by var are scoped to the immediate function body (hence the function scope), whille let variables are scoped to the immediate block.
          • var declared within a function → only available within the function (function-scoped).
          • Does not apply to other code blocks like if, for, while, etc.
        • *Re-declaring & Re-assigning:
          • var lavar = “neva lost” has both actions.
            →
            var : Declaration
            →
            = : Assignment
      • Snake Case is the convention.
        • myVariable = 100;
    • let
      • (def)
        • (mdn) Declares block-scoped local variables. Re-assignable. Unre-declarable.
          • *Block-scoped : (def) The scope created with a pair of curly braces ( { } ).
        • (stack) Difference between var & let: Variables declared by var are scoped to the immediate function body (hence the function scope), whille let variables are scoped to the immediate block.
      • Snake Case is the convention.
        • myVariable = 100;
    • const
      • (def)
        • (mdn) Declares block-scoped local variables. Unre-assignable. Unre-declarable.
          • *constant objects can have properties added/updated/removed.
      • Uppercase Camel Case is the convention.
        • MY_CONSTANT = 100;
